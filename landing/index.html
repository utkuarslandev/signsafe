Everything below is based on how SignSafe currently works: wallet interception, simulation-first analysis, and user-controlled blocking.

Is SignSafe itself safe?

You are right to ask - and we are glad you did. The attack vector is identical. The Crypto Copilot extension used the exact same signTransaction interception technique we use. The difference is intent and verifiability.

SignSafe is fully open source. Every line of code is on GitHub. You can read the content script, the background worker, and the overlay - all three files total less than 400 lines. There is no obfuscation, no backend, no remote code execution.

SignSafe never modifies transactions. It calls Solana's public RPC to simulate, calls AI to explain, then passes the original transaction through unchanged - or throws an error if you block it. We cannot alter what you sign.
Open source No backend Read-only RPC Zero transaction modification

No. SignSafe never has access to your keys, seed phrase, or wallet internals. It only sees what any dApp on the internet already sees: the serialized unsigned transaction that the dApp sends to your wallet for signing.

Your private keys never leave Phantom, Backpack, or whatever wallet you use. They are stored encrypted inside the wallet extension and are never exposed to content scripts like ours. SignSafe intercepts the transaction request, not the signing process itself.

Two external calls are made when you initiate a transaction.

1. simulateTransaction sent to Solana's public RPC endpoint (api.mainnet-beta.solana.com). This is the same call any dApp or wallet makes when estimating fees. It contains no identifying information.

2. The simulation result (balance changes, program IDs, logs) sent to the Claude API for plain-English analysis. No wallet addresses, no private keys, no browsing history - only the on-chain effect of the transaction.

There is no SignSafe backend. No analytics, no telemetry, no user tracking. The extension has no server of its own.

Three permissions: storage (to save your settings), scripting (to inject the proxy into pages), and activeTab (to detect which site is requesting a transaction).

We do not request history, tabs, cookies, bookmarks, or any permission that would let us see what you are doing outside of a transaction request.

How it works

SignSafe adds 1-3 seconds before the wallet popup opens. This is the time it takes to run the RPC simulation and get the AI analysis back.

This delay is intentional. The entire point is to pause before you sign. If a 2-second pause is too slow for your use case, you can disable SignSafe for specific sites from the extension popup.

SignSafe works with wallets that inject a window.solana provider - including Phantom, Backpack, Solflare, and most browser-based Solana wallets.

It does not currently work with hardware wallets used standalone, wallet-connect style mobile signers, or wallets that use non-standard injection interfaces.

Only one wallet extension can hold the primary window.solana slot at a time. If multiple wallet extensions are installed, only the active one is wrapped.

The simulation result from Solana's RPC is ground truth - it cannot be wrong about what the transaction will do. The AI's job is to translate that into plain English and assign a risk level, and that translation can occasionally be over- or under-cautious.

The most common false positive: a legitimate new protocol gets flagged as Review because its program was deployed recently. The most common false negative: a sophisticated drainer that mimics a legitimate transaction structure.

SignSafe is a second opinion, not a guarantee. The final decision is always yours. If something feels wrong even after a green verdict, trust that feeling.

When you click Block, SignSafe throws an error to the dApp - the same error pattern as dismissing a wallet popup. The dApp sees a rejection and the transaction is never broadcast.

SignSafe shows you a breakdown of exactly which instruction triggered the flag, so you can understand what you just avoided. You can optionally submit the malicious site or program address to our community blocklist so other users are protected within the next sync cycle.

Compared to alternatives

Phantom's warnings (powered by Blowfish) are good - and we think you should keep them on. SignSafe is a complementary layer, not a replacement. The key difference: Phantom's warnings appear inside the wallet popup, after the transaction has already reached it. SignSafe intercepts before the popup opens.

More practically: Phantom's integration only covers Phantom. SignSafe wraps any wallet that uses window.solana, including wallets that have no built-in security analysis. Backpack, Solflare, and smaller wallets get the same protection.

Blowfish is an API-based service that wallet providers integrate on their backend. It uses a database of known threats and rule-based heuristics. It is excellent but requires the wallet to have integrated it.

SignSafe uses live on-chain simulation via Solana's public RPC - there is no proprietary database, no subscription, and no wallet integration required. The simulation tells you exactly what will happen, not just whether it matches a known pattern. A brand-new drainer that Blowfish has not seen yet can still be caught by SignSafe if the simulation shows tokens draining.

Free for personal use, always. The extension costs nothing and has no usage limits.

The API costs (Solana RPC + Claude AI) are negligible per transaction - under $0.01 per analysis at current pricing. For the current beta, these are covered by the team.

The long-term model is a developer API and SDK for dApps and wallet providers that want to embed SignSafe analysis natively, while the personal extension stays free.

Edge cases

Two protections: first, SignSafe is open source so any version can be diffed against the previous one on GitHub. Community review of updates is encouraged. Second, the extension permissions are intentionally minimal, so even a compromised version cannot access your keys or browsing history.

If you want to be certain: pin the extension version in Chrome and only update after the community has reviewed the new commit.

Not currently. SignSafe is a desktop Chrome extension and cannot hook into mobile wallet in-app browser architectures yet.

If you primarily use Solana on mobile, the best current option is to use Phantom's built-in transaction screening and treat any transaction link from a DM, tweet, or Discord message with high suspicion before approving.

If RPC simulation fails for any reason - network error, timeout, expired blockhash - SignSafe defaults to a Review verdict with a clear notice that analysis was incomplete. It does not silently pass the transaction through as Safe.

You always retain the choice to proceed or block. A failed simulation means less information, not a green light.
